# CRITERIOS DE ACEPTACIÓN DE MÓDULOS  
## Gate Clínico de Integración  
**Versión 1.0**

---

## 1. Propósito del documento

Establecer los requisitos mínimos que debe cumplir cualquier módulo, herramienta o función para ser integrada al sistema.

Este documento actúa como **filtro clínico obligatorio**.

---

## 2. Criterios obligatorios (todos deben cumplirse)

### 2.1 Duración
- El uso típico en sesión no debe exceder **2–3 minutos**.

### 2.2 Control
- El inicio y fin del ejercicio deben depender del clínico.
- El paciente no decide cuándo “termina” terapéuticamente.

### 2.3 Resultado
- No debe prometer alivio, calma ni mejoría emocional.
- El foco es la relación con la experiencia.

### 2.4 Lenguaje
- No usar términos de control emocional.
- No usar lenguaje motivacional ni prescriptivo.

### 2.5 Abandono seguro (fallback)
- El ejercicio puede interrumpirse sin consecuencias.
- No existe “fracaso” si no se completa.

---

## 3. Criterios de exclusión inmediata

Un módulo se rechaza automáticamente si:

- Genera puntuaciones clínicas.
- Compara sesiones pasadas.
- Ofrece recomendaciones automáticas.
- Evalúa progreso, riesgo o severidad.
- Funciona sin mediación clínica.

---

## 4. Validación previa a integración

Antes de integrar un módulo se debe responder afirmativamente:

1. ¿Facilita contacto funcional?
2. ¿Permite elección consciente?
3. ¿Evita control encubierto?

Si alguna respuesta es “no”, el módulo no se integra.

---

## 5. Estado del documento

Este archivo es normativo y vinculante.  
No admite excepciones implícitas.
